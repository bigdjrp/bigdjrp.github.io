Why not a Recursive Algorithm?

With regards to using a recursive algorithm, and it's use to compute the Fibonacci sequence, recursion may useful for a small amount of integers. Because the computational complexity of this algorithm is 0(n^2), the more input, the longer the program takes to complete.
  
  











//

# include <stdio.h>

int main(){
int i, current;
int n;
int node1 = 0;
int node2 = 1;
printf("Enter a positive integer: ");
scanf("%d", &n);
if(n == 0){
printf("\n0");
}
else if(n == 1){
printf("\n0 \n1");
}
else{
printf("\n0 \n");
}

for(i = 0; i <= n-2; i++){
current = node1 + node2;
printf("%d \n", current);
node2 = node1;
node1 = current;
}
}
